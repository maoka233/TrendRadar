# TrendRadar 配置说明

## 当前配置概览

本系统已针对特定需求进行了优化配置。

### 监控平台 (共4个)

系统现仅关注以下4个平台的热点话题：

1. **微博** - 社交媒体热点
2. **贴吧** - 论坛讨论热点  
3. **知乎** - 知识问答热点
4. **bilibili** - 视频弹幕热点

### 关键词配置 (共7组)

系统会从以上4个平台筛选出以下话题相关的热点：

#### 1. AI相关
- ai / AI / 人工智能

#### 2. GPT相关  
- GPT / chatgpt / gpt

#### 3. Claude相关
- claude / Claude

#### 4. DeepSeek相关
- deepseek / DeepSeek

#### 5. 新能源汽车相关
- 新能源汽车 / 电动车 / 新能源车

#### 6. 手机相关
- 手机 / 智能手机

#### 7. 芯片相关
- 芯片 / 半导体 / 处理器

## 使用说明

### 快速开始

1. **查看热点报告**
   - HTML报告位置: `output/当前日期/html/当日汇总.html`
   - 或查看项目根目录的 `index.html`

2. **推送通知** (可选)
   - 在 `config/config.yaml` 中配置通知渠道
   - 或使用 GitHub Secrets 配置（推荐）

### 热点展示内容

系统为每个热点话题提供：

- ✅ **话题标题** - 清晰展示话题内容
- ✅ **平台来源** - 标注来自哪个平台
- ✅ **热度指标**
  - 排名历史（最高/最低排名）
  - 出现频次（监控期间出现次数）
  - 持续时间（首次到最后出现的时间跨度）
- ✅ **话题链接** - 点击可跳转查看原文和评论
- ✅ **新增标记** - 🆕标记新出现的话题

### 报告模式

系统支持3种报告模式（在 `config/config.yaml` 中配置）：

1. **daily** (当日汇总) - 默认
   - 显示当日所有匹配的热点
   - 按时推送（如每小时一次）
   - 适合：全面了解当日热点趋势

2. **current** (当前榜单)
   - 显示当前最新的榜单热点
   - 按时推送
   - 适合：追踪实时热点

3. **incremental** (增量监控)
   - 仅显示新增的热点
   - 有新增时才推送
   - 适合：避免重复信息

## 自定义配置

### 修改监控平台

编辑 `config/config.yaml` 的 `platforms` 部分：

```yaml
platforms:
  - id: "weibo"
    name: "微博"
  - id: "tieba"
    name: "贴吧"
  # 添加或删除平台...
```

可用平台ID参考：[newsnow项目源代码](https://github.com/ourongxing/newsnow/tree/main/server/sources)

### 修改关键词

编辑 `config/frequency_words.txt`：

```
关键词1
关键词2
关键词3

另一组关键词1
另一组关键词2
```

**注意**：
- 使用空行分隔不同的关键词组
- 每组内的关键词会被视为"或"关系
- 不同组之间独立统计

## 高级功能

### 关键词语法

支持三种特殊语法：

1. **普通词** - 直接写关键词
   ```
   AI
   手机
   ```

2. **必须词** - 以 `+` 开头，必须同时出现
   ```
   AI
   +技术
   ```
   效果：只匹配同时包含"AI"和"技术"的标题

3. **过滤词** - 以 `!` 开头，排除包含该词的标题
   ```
   AI
   !广告
   ```
   效果：匹配"AI"但排除包含"广告"的标题

### 权重调整

在 `config/config.yaml` 中调整权重算法：

```yaml
weight:
  rank_weight: 0.6      # 排名权重 (0-1)
  frequency_weight: 0.3 # 频次权重 (0-1)  
  hotness_weight: 0.1   # 热度权重 (0-1)
```

三个权重加起来必须等于1.0

- 提高 `rank_weight` → 更关注排名高的新闻
- 提高 `frequency_weight` → 更关注反复出现的新闻
- 提高 `hotness_weight` → 更关注持续在高位的新闻

## 常见问题

### Q: 为什么没有评论内容？

A: 当前数据源（newsnow API）只提供话题标题和链接，不包含评论。要查看评论，请点击话题链接跳转到原平台。

### Q: 如何查看更多平台？

A: 访问 [newsnow网站](https://newsnow.busiyi.world/)，点击"更多"查看支持的平台，然后在 `config/config.yaml` 中添加对应的平台ID。

### Q: 如何调整推送频率？

A: 编辑 `.github/workflows/crawler.yml` 中的 `cron` 表达式，或使用Docker部署时设置 `CRON_SCHEDULE` 环境变量。

### Q: 热点太多/太少怎么办？

A: 
- 太多：增加更具体的关键词，使用必须词(+)语法
- 太少：增加相关的近义词，检查关键词拼写
